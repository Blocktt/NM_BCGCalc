A single zip file will be available for download. It will include all result files, reference files, and the original input data file. All result data files will be CSV format.

The zip file will included the base name of the original file along with the date and time.

The individual files and their contents are output in three subfolders. Each output file is described in the table below.

File Builder, Taxa Translator, zip file contents. ‘xxxx’ represents either ‘bugs’ or ‘fish’ depending on chosen OTU. ‘yyyymmdd’ represents the date when the associated file has last been updated.
Output file Subfolder Description
Original file name _user_input Original input file
NM_BCG_xxxx_Attributes_yyyymmdd reference Regional attribute table.
NM_BCG_xxxx_Attributes_Metadata_yyyymmdd reference Metadata for the regional attribute table
NM_BCG_xxxx_Taxa_Translator_yyyymmdd reference Regional taxa translator table
NM_BCG_xxxx_Taxa_Translator_Metadata_yyyymmdd reference Metadata for the regional taxa translator table
BCG_TaxaTranslator_modify results_xxxx_BCG Lists the taxa that occur in the input file, plus: 1) original TaxonID; 2) ‘translated’ TaxonID (OTU that will be used for the calculation); 3) whether the original TaxonID occurs in the regional taxa translator table (TRUE/FALSE); 4) whether the original TaxonID was changed during the ‘translation’ process (YES/NO); 5) total count in the input file (# individuals summed across samples)
BCG_TaxaTranslator_nonmatch results_xxxx_BCG Lists taxa in the input file that don’t occur in the regional taxa translator table (if the file is blank, all are ‘matches’)
BCG_TaxaTranslator_source results_xxxx_BCG Provides documentation of the Operational Taxonomic Unit (OTU) and source files that were used
BCG_TaxaTranslator_TAXAATTR results_xxxx_BCG Final output, with ‘translated’ TaxonID, NonTarget designations, attributes and phylogeny; the file is ready to be run through the BCG calculator
